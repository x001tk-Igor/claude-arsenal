# CONTRIBUTING.md — как создавать skill или агент в claude-arsenal

Этот гайд — для тебя и для любого контрибьютора. Если ты создаёшь skill/агент в этом репо — прочти сначала.

---

## Главный принцип

> **Skill = 3 слоя. Description → Instructions → Tools. Без tools — это не skill, а промт.**

Skill с пустым `tools/` проигрывает по качеству, стабильности и стоимости. Всегда заполняй третий слой.

---

## Структура skill'а (обязательная)

```
skills/<имя-скилла>/
├── SKILL.md            # description + instructions
├── tools/              # слой 3 — детерминированный код
│   ├── *.py / *.sh     # скрипты
│   ├── template.md     # шаблоны вывода
│   └── *.json          # конфиги, справочники
├── examples/           # few-shot: эталонные примеры
│   └── example-1.md
└── README.md           # (опционально) для конечного пользователя
```

---

## Шаблон SKILL.md

```yaml
---
name: <kebab-case-имя>
description: <ТОЧНОЕ "когда брать этот skill">. Trigger when user says: "..." or "...".
---

# <Имя skill>

## Назначение
<1-2 абзаца: что делает и зачем>

## Когда использовать
- <триггер 1>
- <триггер 2>
- <триггер 3>

## Процесс работы
1. <шаг>
2. <шаг>
3. <шаг>

## Инструменты (tools/)
- `tools/<скрипт>.py` — <что делает>
- `tools/<шаблон>.md` — <когда подставлять>

## Примеры
См. `examples/`

## Правила
- <правило 1>
- <правило 2>
```

## Правила хорошего description

Description — это **главный триггер**. Skill выбирается по нему, а не по имени.

**Хорошо:**
> "Парсит вакансии с hh.ru в таблицу. Use when user mentions hh.ru, headhunter, вакансии, зарплаты, or wants to collect job listings into a table."

**Плохо:**
> "Useful for working with job sites and data extraction."

Размытое description = skill не активируется.

**Формула:**
1. Что делает (1 предложение)
2. Когда брать (use when...)
3. Триггер-слова на языках пользователя (RU + EN)

## Когда создавать новый skill

Создавай новый skill, если:
- Паттерн повторяется 2+ раза
- Логика детерминированная (можно вынести в `tools/`)
- Skill узкий (одна задача, не «всё-в-одном»)

**НЕ создавай skill, если:**
- Задача уникальная и не повторится
- Это просто «удобная команда» — лучше сделай alias в `tools/`
- Skill будет делать 10 разных вещей — разбей на 3-5 узких

---

## Композиция vs монолит

**Плохо:** `content-creation` (внутри: research + copy + hooks + SEO + форматирование).
- Сломалось — непонятно где
- Улучшил один кейс — поломал три
- Невозможно тестировать изолированно

**Хорошо:** `research` · `draft` · `headlines` — каждый тестируется и улучшается отдельно.

Правило: **один skill — одна ответственность**. Если не можешь описать skill в одном предложении — разбей.

## PR / commit в claude-arsenal

1. Создай ветку: `git checkout -b skill/<имя>`
2. Положи skill в `skills/<имя>/` или агента в `agents/<имя>.md`
3. Обнови `docs/SKILL_INDEX.md` (если добавляешь skill)
4. Проверь, что `tools/` заполнен
5. Commit, push, открой PR

Формат commit message:
```
[skill] add <имя> — <что делает одной строкой>
[agent] add <имя> — <что делает одной строкой>
[fix] <имя> — <что починил>
[docs] <что обновил>
```
