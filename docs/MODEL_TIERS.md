# MODEL_TIERS — какую модель когда звать

Главное правило из bonus-workflows: **opus — только на финал**. Всё остальное — haiku/sonnet. Это съедает 70-80% бюджета на умных моделях.

## Карта применения

| Фаза воркфлоу | Модель | Почему |
|---|---|---|
| Роутинг / scope / разложение на углы | **haiku** | Один структурированный JSON на выходе, не нужен «интеллект» |
| Извлечение по тексту: болей, цитат, тем, тональности | **haiku** | Ротина, много кусков, шаблонная разметка |
| Фанат-аут: один агент на объект (конкурент, файл, угол) | **sonnet** | Нужно качество, но задача локальная и ограниченная |
| Верификация / судьи (3-judge panel, adversarial verify) | **sonnet** | Один и тот же skill, разные «роли» — sonnet справляется |
| **Финальный synthesis в Markdown-отчёт** | **opus** | Сводит N источников, пишет связный текст — это «дорого, но один раз» |

## Паттерн экономии (из bonus-workflows)

```js
// ПЛОХО: opus на каждой фазе — сжигает бюджет
const a = await agent(prompt, { model: 'opus' })
const b = await agent(prompt, { model: 'opus' })
const c = await agent(prompt, { model: 'opus' })

// ХОРОШО: haiku/sonnet веером, opus ровно один раз в конце
const raw = await parallel(items.map(i => () =>
  agent(prompt, { model: 'sonnet', schema: X })   // fan-out
))
const report = await agent(synthPrompt, { model: 'opus' })  // 1 вызов
```

## Ограничители расхода (из bonus-workflows)

Когда данных может быть очень много:

```js
const MAX = 80            // потолок на число объектов (файлов, источников)
const CHUNK = 20          // размер одного куска
const MAX_CHUNKS = 40     // потолок на число кусков (800 записей max)
```

Плюс встроенный лимит системы: **≤16 параллельных агентов, ≤1000 за запуск**.

## Свой budget

В Dynamic Workflows есть `budget.remaining()` — можно динамически выходить из цикла:

```js
while (budget.remaining() > 50_000) {
  // …набираем findings
}
if (!budget.total) {
  // безлимит — спиннинг может уйти в долгое копание
  // ограничь while счётчиком
}
```

## Шпаргалка выбора

| Если задача… | Бери модель |
|---|---|
| Извлечь JSON-поле из текста по схеме | haiku |
| Найти и перечислить факты с источниками | sonnet |
| Оценить идею по 3 критериям (scoring) | sonnet |
| Свести 10-50 разборов в связный отчёт | opus |
| Принять финальное решение (top-3, рекомендации) | opus |

**Эвристика:** если у выхода есть **JSON-схема** — почти всегда хватит sonnet. Если выход — **связный текст для человека** — opus.
