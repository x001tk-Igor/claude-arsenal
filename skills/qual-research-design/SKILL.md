# Дизайн качественного полевого исследования

Скилл ведёт через 4 шага: сбор контекста → внутренний анализ → дизайн с разбивкой по сегментам и методам → финальная таблица.

Правила выбора метода — в [METHOD-RULES.md](METHOD-RULES.md). Загрузить перед шагом 2.

---

## Шаг 1 — Сбор контекста

Попросить пользователя загрузить всё, что есть по проекту. Одним сообщением запросить:

1. **Продукт/бренд клиента** — что за продукт, категория, ценовой сегмент
2. **Бизнес-задача и контекст** — зачем исследование, какое решение должно быть принято по итогам
3. **Сегменты или гипотезы сегментов ЦА** — кого исследуем (если есть)
4. **География** — города, регионы
5. **Задачи на исследование** — что конкретно нужно узнать (если сформулированы)
6. **Бюджетные ограничения** — есть ли потолок по количеству интервью/групп

Если чего-то не хватает — переспросить. Минимум для старта: продукт + бизнес-задача + география.

Не показывать меню с вариантами. Просто попросить загрузить контекст.

---

## Шаг 1б — Определение ключевой переменной сегментации (ПОКАЗАТЬ пользователю)

После получения контекста — определить, **по какому признаку респонденты должны принципиально отличаться друг от друга**, исходя из бизнес-задачи. Это главная ось, вокруг которой строится вся выборка.

Сегменты формируются по переменной, которая является **центральной для исследовательского вопроса**, а не по демографии или удобству.

### Как определить ключевую переменную

Задать себе вопрос: **«Какое различие между людьми даст нам ответ на бизнес-задачу?»**

Типичные переменные в зависимости от задачи:

| Бизнес-задача | Ключевая переменная | Пример |
|---|---|---|
| Бренд-стратегия, ребрендинг | **Отношение к бренду** (лояльность) | Лояльные / нелояльные / свитчеры |
| Позиционирование | **Восприятие категории и брендов** | По-разному воспринимают ценность продукта |
| Запуск нового продукта | **Процесс покупки** в категории | Покупают онлайн / офлайн / импульсно / планово |
| Коммуникационная стратегия | **Медиаповедение и точки контакта** | Где узнают о продуктах, чему доверяют |
| Улучшение продукта | **Способ использования** продукта | Ежедневно / от случая к случаю / по особым поводам |
| Выход в новый сегмент | **Опыт в категории** | Новички / опытные / эксперты |
| Конкурентный анализ | **Предпочитаемый бренд/решение** | Пользователи бренда А / Б / В |

### Что делать

1. На основе бизнес-задачи и контекста сформулировать **гипотезу ключевой переменной** — что именно должно отличать респондентов между собой
2. Предложить пользователю: «Исходя из задачи, предлагаю строить выборку вокруг [переменная], потому что [обоснование]. Демографию (пол, возраст) используем как вторую ось для сравнения. Согласны, или есть другой ключевой признак?»
3. Дождаться подтверждения или корректировки

**Важно:** демография (пол, возраст, доход) — это **второстепенная ось**. Она добавляется для сравнения внутри ключевой переменной, но не определяет структуру исследования.

## Шаг 2 — Внутренний анализ (НЕ показывать пользователю)

Прочитать [METHOD-RULES.md](METHOD-RULES.md), затем для каждого сегмента оценить:

### 2а. Оценка темы

| Фактор | Варианты | Влияние |
|---|---|---|
| **Сенситивность темы** | Низкая / Средняя / Высокая | Высокая → только IDI, нельзя ФГ |
| **Тип задачи** | Бренд-стратегия / Коммуникация / CJM / Концепт-тест / Позиционирование | CJM → только IDI; концепт-тест → можно ФГ |
| **Нужна ли групповая динамика** | Да / Нет | Да → ФГ подходит; Нет → IDI |

### 2б. Оценка аудитории (по каждому сегменту)

| Фактор | Варианты | Влияние |
|---|---|---|
| **Тип аудитории** | Массовая / Премиум-люкс / C-level B2B / Узкие специалисты | Премиум/C-level/узкие → экспертные интервью |
| **Географическая распределённость** | Один город / Несколько городов / Разброс | Разброс → IDI (онлайн) |
| **Однородность сегмента** | Высокая / Средняя / Низкая | Высокая → ФГ возможна; Низкая → IDI |

### 2в. Нужно ли деление лояльные/нелояльные

Включать, когда:
- Задача — бренд-стратегия существующего бренда
- Задача — репозиционирование
- Задача — понять, почему уходят к конкурентам
- Есть явное указание на конкурентов

НЕ включать, когда:
- Запуск нового бренда с нуля
- Исследование категории без привязки к бренду
- Тест концепций/коммуникаций до запуска

### 2г. Определение метода по сегменту

Применить дерево решений из METHOD-RULES.md. Каждый сегмент получает один из трёх методов:
- **IDI** (глубинное интервью) — базовый метод
- **ФГ** (фокус-группа) — если тема не сенситивная, аудитория массовая и однородная, нужна групповая динамика
- **Экспертное интервью** — если аудитория премиум/люкс, C-level B2B, узкие специалисты

### 2д. Расчёт количества

**IDI:**
- Базовое: 5–7 интервью на гомогенный сегмент (насыщение)
- Минимум при ограничениях: 4
- Если сегмент неоднородный (широкий возрастной разброс, разная география): +2–3
- Для второстепенных сегментов: можно 2–3 (достаточно для обнаружения паттернов, но не для насыщения — указать это)

**ФГ (5–6 человек в стандартной группе):**
- **Маркетинговое исследование** (отдельный проект, нормальный бюджет): от 3 групп на сегмент
- **Бренд-стратегия / ребрендинг / бренд-платформа**: 2 группы на сегмент (допустимо меньше — исследование входит в более широкий проект)
- Мини-группа (3–4 чел.) — для узких сегментов, сложного рекрута
- Диада (2 чел.) — глубина IDI + минимальная групповая динамика (контрастные пары, сложный рекрут)
- Если несколько городов: минимум 1 группа на город
- Минимум при ограничениях: 2 группы
- **Двухуровневая ФГ**: если нужна реакция «второго уровня» (напр. сначала потребители, потом бренд-команда клиента обсуждает результаты)

**Экспертные интервью:**
- Базовое: 3–5 на сегмент
- Минимум: 2

При бюджетных ограничениях — сначала урезать второстепенные сегменты, потом уменьшать до минимума по основным.

## Шаг 3 — Вывод пользователю: дизайн исследования

### 3а. Рекомендация по методу

Объяснить, какой метод рекомендуется и **почему** (1–3 предложения). Если разные сегменты требуют разных методов — объяснить для каждого.

Подсветить ограничения:
- Если тема сенситивная → объяснить, почему ФГ не подходит
- Если аудитория труднодостижимая → объяснить, почему экспертные
- Если задача — CJM → объяснить, почему только IDI

### 3б. Разбивка по сегментам

Формат вывода — иерархический список с итогами:

```
### [Название сегментной группы], всего [N] интервью

– [Подсегмент 1: лояльность + демография/характеристика] — [N] интервью
  Обоснование: [1 предложение — почему столько и зачем этот подсегмент]
– [Подсегмент 2] — [N] интервью
  Обоснование: [...]

### [Следующая группа], всего [N] интервью
...

---
**Итого: [N] интервью / [N] фокус-групп / [N] экспертных**
```

Пример:

```
### Женщины, всего 10 интервью

– Нелояльные, частые покупатели магазинов-конкурентов, 25–35 лет — 3 интервью
  Обоснование: ключевой сегмент для понимания барьеров переключения на бренд
– Нелояльные, 35+ лет — 2 интервью
  Обоснование: второстепенный сегмент, достаточно для обнаружения отличий от молодой аудитории
– Лояльные покупатели бренда, 25–35 лет — 3 интервью
  Обоснование: понять драйверы лояльности в молодой аудитории
– Лояльные покупатели бренда, 35+ лет — 2 интервью
  Обоснование: контроль — сравнить мотивацию с молодыми лояльными

### Мужчины, всего 2 интервью

– Нелояльные, 25–35 лет — 1 интервью
  Обоснование: проверочный сегмент — отличается ли мужская аудитория принципиально
– Лояльные, 35+ лет — 1 интервью
  Обоснование: минимальное покрытие для сравнения
```

### 3в. Комментарии и оговорки

Добавить, если применимо:
- Какие сегменты можно сократить первыми при урезании бюджета
- Где 2–3 интервью достаточно только для обнаружения паттернов, но не для насыщения
- Если предлагается микс методов (IDI + ФГ) — объяснить логику распределения
