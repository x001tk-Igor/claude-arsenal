# Промпты трёх параллельных агентов

Каждый агент самостоятельно исследует свой блок и сохраняет результат через `Write`. Базовые переменные: `{BRAND}`, `{SITE}`, `{CATEGORY}`, `{OUT_DIR}` — итоговая папка `/Users/fixed/Documents/ZB CLOUD 3600/CLOUD ZB (3600)/Researches/[slug]/`.

Все три `Agent`-вызова отправлять в **одном сообщении** (один блок tool-calls), чтобы исполнялись параллельно.

---

## Общие требования (добавлять в каждый промпт)

- Русский язык в файле
- Под каждым утверждением — цитата и/или URL источника
- Факты — без пометки, догадки и интерпретации — с пометкой `[вывод]`
- Если данных мало — писать «данных недостаточно», не выдумывать
- Сохранить файл через `Write` по указанному пути (не возвращать полный текст в ответе)
- Вернуть в ответ резюме 5–10 строк: что нашёл, где были дыры

## Агент 1 — Продукты и сайт

```
description: "Brand: сайт и продукты"
subagent_type: "general-purpose"
prompt:

Ты собираешь данные о бренде "{BRAND}" ({SITE}) для последующего бренд-анализа. Работаешь с САЙТОМ и продуктовыми материалами. Параллельно другие агенты собирают соцсети и рекламу — не пересекайся с ними.

Задача:
1. WebFetch главной страницы {SITE} — слоган, теглайн, главные сообщения, навигация, как обращаются к посетителю.
2. WebFetch страницы «О компании» / «О нас» / «Миссия» / «Философия» — миссия, видение, ценности, история.
3. WebFetch 2–4 ключевых продуктовых страниц — описания продуктов, УТП, RTB, для кого, цена.
4. Если есть страницы «Карьера» или «Пресса» — иногда там лучшая формулировка миссии.
5. Собрать описание заявленной ЦА — кому продают, какой язык используют.

Оформи файл по структуре:

# 01. Продукты и сайт — {BRAND}

**Дата:** {сегодня}
**Источник:** {SITE}

## Продукты
Таблица: название | короткое описание (1 строка) | цена | ценовой сегмент (бюджетный/middle/upper-middle/premium) | для кого.

## Ключевые сообщения с сайта
Буквальные цитаты с URL-ссылкой где нашёл.

## Слоганы, теглайны
Копипаст.

## Миссия / видение / большая цель
Буквальные цитаты. Если не сформулированы — написать «не сформулировано на сайте».

## Как бренд описывает себя
5–10 прямых цитат про сам бренд с сайта («мы —…», «мы помогаем…»).

## Заявленная ЦА с сайта
Цитаты и описания, кого они называют своей аудиторией.

## Тон сайта
Несколько примеров формулировок с сайта, чтобы видно было как он общается (на «вы»/«ты», формальный/дружеский и т.д.).

Сохрани в: {OUT_DIR}/01_products_website.md
Верни резюме на 5–10 строк.
```

## Агент 2 — Соцсети и контент

```
description: "Brand: соцсети и посты"
subagent_type: "general-purpose"
prompt:

Ты собираешь посты и контент из соцсетей бренда "{BRAND}" для бренд-анализа. Параллельно другие агенты работают с сайтом и рекламой.

Задача:
1. Найти соцсети бренда — сначала WebFetch футера сайта {SITE}, оттуда взять ссылки на Telegram / VK / YouTube / Instagram / RuTube. Не угадывать URL. Если в футере нет — WebSearch `"{BRAND}" соцсети`.
2. **Описания каналов — ОБЯЗАТЕЛЬНО.** Для каждой соцсети скопировать:
   - VK: описание группы (раздел «Информация»)
   - Telegram: bio/описание канала (отображается в шапке, доступно через https://t.me/s/канал)
   - YouTube: раздел «О канале»
   - Instagram: bio из шапки профиля (если публично доступно через поиск/WebSearch сниппеты)
   Это критично: «как бренд пишет про себя» — один из самых важных источников для позиционирования.
3. WebFetch публичных страниц Telegram (https://t.me/s/канал) и VK — собрать ~15 последних постов.
4. Для каждого поста записать: дата, первые 400–600 символов текста, ссылку на пост.
5. WebSearch "{BRAND}" — посмотреть как бренд описан в сниппетах поисковика (часто это meta description, он тоже формирует восприятие).

Оформи файл по структуре:

# 02. Соцсети и контент — {BRAND}

**Дата:** {сегодня}

## Список каналов (взято с сайта)
Канал | URL.

## Описания каналов (как бренд пишет про себя)
Для каждой соцсети — копипаст описания из шапки/bio + URL. Если не нашёл — написать «не удалось получить».

## Посты (~15 штук)
Для каждого:
### [Дата] [Платформа]
Ссылка на пост.
Текст поста (400–600 символов или полностью, если короче).
Короткий комментарий [вывод]: о чём пост, какое обращение к аудитории, какой тон.

## Темы контента
Перечень повторяющихся тем по частоте.

## Герои постов
Кого показывают в контенте (клиенты, эксперты, сотрудники, CEO, продукт сам по себе).

## Обращение к аудитории
На «ты»/«вы», использование сленга, эмодзи, мемов.

## Сниппеты в поиске
Что выдаёт Google/Яндекс про бренд в описаниях сайтов.

Сохрани в: {OUT_DIR}/02_social_content.md
Верни резюме на 5–10 строк.
```

## Агент 3 — Реклама, интеграции и PR

Методы взяты из конкретного подхода пользователя. Главный хак — маркер `erid` (обязательная маркировка рекламы в РФ), по нему отсеивается именно платное размещение, а не органика.

**Что ищем:** интеграции в Telegram / YouTube / Instagram через `erid`, лендинги (прокликивая рекламу), спецпроекты бренда, наружка (опционально), коммерческие PR-истории (кейсы, а не пресс-релизы), интервью основателей.
**Что НЕ ищем:** платные сервисы (Medialogia, tgstat, trendhero, barkov.net, Semrush, Similarweb), Яндекс.Дзен, ТВ-ролики, FB Ads Library / Google Ads Transparency / TikTok / LinkedIn (отдельная работа, не в этом скилле). Paid search и organic search через сервисы — не смотрим, только если случайно попадутся в выдаче.

```
description: "Brand: реклама, интеграции, PR"
subagent_type: "general-purpose"
prompt:

Ты собираешь рекламные интеграции и PR бренда "{BRAND}" для бренд-анализа. Параллельно другие агенты работают с сайтом и соцсетями.

Главный метод — искать по маркеру ERID (в РФ реклама маркируется `erid: ...`). Это отсеивает именно оплаченные размещения.

Задача:

### 1. Интеграции в Telegram
- WebSearch `"{BRAND}" erid site:t.me` — находим посты с маркированной рекламой
- WebSearch `"{BRAND}" erid telegram` — альтернативная формулировка
- Для каждого поста: канал, дата, суть поста, ссылка
- Приоритет — каналы с большими охватами (если видно по контексту)

### 2. Интеграции в YouTube
- WebSearch `site:youtube.com erid "{BRAND}"` — ролики с маркировкой
- WebSearch `"{BRAND}" интеграция youtube` как дополнительный
- Для каждого: канал блогера, название видео, суть интеграции, ссылка

### 3. Интеграции в Instagram
- WebSearch `"{BRAND}" erid instagram` + `"{BRAND}" инстаграм реклама`
- Для каждого: блогер, суть, ссылка (если публично)

### 4. Лендинги продуктов
- По ссылкам из найденной рекламы — куда ведут (часто это отдельные поддомены/лендинги, не индексируемые с главного сайта)
- WebFetch 1–3 таких лендингов — зафиксировать главный оффер, УТП, кому адресовано

### 5. Спецпроекты и коллаборации
- WebSearch `"{BRAND}" спецпроект`
- WebSearch `"{BRAND}" коллаборация OR коллаб`
- WebSearch `"{BRAND}" site:vc.ru` (часто разборы спецпроектов)
- Кратко: что за проект, с кем, идея

### 6. Наружная реклама (опционально)
- WebSearch `"{BRAND}" наружная реклама` + Google Images — если быстро нашлось, описать визуал и сообщение, дать ссылку. Если с ходу нет — пропустить, не копать глубоко.

### 7. PR и СМИ — только КОММЕРЧЕСКИЕ истории
- Ищем разборы рекламных кейсов и маркетинговых проектов, НЕ пресс-релизы и новости компании:
  - WebSearch `"{BRAND}" кейс site:sostav.ru`
  - WebSearch `"{BRAND}" кейс site:adindex.ru`
  - WebSearch `"{BRAND}" маркетинг site:vc.ru`
  - WebSearch `"{BRAND}" site:cossa.ru`
- Отфильтровать: берём материалы про маркетинг/коммуникацию/бренд; не берём «компания открыла новый офис» и «выручка выросла на 20%».

### 8. Интервью основателей и CMO
- WebSearch `"{BRAND}" интервью основатель OR CEO OR CMO`
- Цитаты про миссию, позиционирование, ЦА, стратегию — с датой и ссылкой
- Опционально Exa для свежих материалов с фильтром по датам

Оформи файл по структуре:

# 03. Реклама, интеграции и PR — {BRAND}

**Дата:** {сегодня}

## Интеграции в Telegram
Таблица: канал | дата | суть | ссылка. Под таблицей [вывод]: о чём чаще всего говорят в интеграциях, какое сообщение бренда транслируют.

## Интеграции в YouTube
Таблица: канал блогера | название ролика | суть | ссылка. Под таблицей [вывод].

## Интеграции в Instagram
Таблица: блогер | суть | ссылка. Если данных нет — «данных недостаточно».

## Лендинги продуктов
Для каждого: URL, главный оффер, УТП, на кого таргет. Указать откуда перешёл (из какой рекламы).

## Спецпроекты и коллаборации
Название, партнёр, идея, год, ссылка на разбор.

## Наружная реклама
Если нашёл — визуал (описание), сообщение, ссылка. Если не нашёл — пропустить.

## Коммерческие PR-истории
Разборы кейсов и маркетинговых проектов. Заголовок, короткий пересказ, ссылка.

## Цитаты из интервью основателей и топов
Буквальные цитаты про миссию, позиционирование, ЦА, стратегию. Формат: `«цитата» — Имя, должность, издание, дата, [ссылка]`.

Сохрани в: {OUT_DIR}/03_advertising_pr.md
Верни резюме на 5–10 строк: сколько интеграций нашёл, главные темы в рекламе, какие дыры остались.
```
