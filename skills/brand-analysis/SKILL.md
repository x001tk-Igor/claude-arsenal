# Brand Analysis

Глубокий разбор одного бренда: позиционирование, продукт, коммуникация. Три параллельных агента собирают сырьё → один файл с выводами.

## Инпут

Спросить у пользователя (если не указано):
1. Название бренда и сайт (если знает)
2. Рынок/категория (для контекста, опционально)

Если в `company-profile-search/output/` уже есть профиль компании — подхватить базовые факты (продукты, цены), не собирать заново.

## Шаг 1. Создать папку проекта

Slug = латиница, нижний регистр, дефисы. Например: `tinkoff`, `yandex-lavka`.

```
/Users/fixed/Documents/ZB CLOUD 3600/CLOUD ZB (3600)/Researches/[slug]/
```

Внутри — 4 файла: три от агентов + один итоговый синтез.

## Шаг 2. Запустить 3 параллельных агента

**ВАЖНО:** все три вызова `Agent` — в одном сообщении (один блок tool-calls), `subagent_type="general-purpose"`. Каждый агент сам пишет результат в свой файл через `Write` и возвращает короткое резюме.

Полные промпты агентов: [references/agent-prompts.md](references/agent-prompts.md). Общие требования для всех трёх:
- Цитаты-доказательства под каждым утверждением
- Ссылки (URL) под каждым блоком — откуда взято
- Русский язык; факты — без пометки, догадки — с пометкой `[вывод]`
- Сохранить файл через `Write` по указанному пути
- В конце вернуть 5–10 строк резюме того, что собрал

**Агент 1 — Продукты и сайт** → `01_products_website.md`
Главная, «О нас», продуктовые страницы, миссия. Продуктовая линейка, цены и ценовой сегмент, теглайны, слоганы, ключевые сообщения с сайта, заявленная ЦА.

**Агент 2 — Соцсети и контент** → `02_social_content.md`
~15 последних постов из Telegram/VK через `WebFetch` (текст + дата + ссылка). **Обязательно: описания/bio каналов** (как бренд пишет про себя — VK, Telegram, YouTube, Instagram). Темы, тон, герои постов, сниппеты бренда из поиска.

**Агент 3 — Реклама, интеграции и PR** → `03_advertising_pr.md`
Главный метод — поиск по маркеру `erid` (маркировка рекламы в РФ): интеграции в Telegram, YouTube, Instagram. Лендинги продуктов (по ссылкам из найденной рекламы). Спецпроекты и коллаборации. Наружка — опционально. PR — только коммерческие истории/кейсы (sostav.ru, adindex.ru, vc.ru, cossa.ru), не пресс-релизы. Интервью основателей про миссию/позиционирование. **Не смотрим:** ТВ-ролики, Яндекс.Дзен, платные сервисы, FB/Google Ads библиотеки.

## Шаг 3. Синтез → итоговый файл

Прочитать все 3 файла и написать `brand_analysis.md` по шаблону [references/output-template.md](references/output-template.md).

Алгоритм анализа (в этом порядке):

1. Продукты, короткое описание, ценовой сегмент
2. Сводка собранной текстовой коммуникации (сайт, соцсети, сниппеты)
3. Сводка примеров рекламы
4. Гипотезы ЦА — из сайта, контента, продуктов (с логикой вывода)
5. Ключевые сообщения и теглайны
6. Герои и контекст коммуникации — позиция бренда относительно ЦА (наставник / свой / эксперт / помощник / друг / провайдер)
7. Характер бренда и ToV (формальный / фановый / про полезность / эмоциональный / экспертный) + 2–3 цитаты-доказательства
8. Попытки отличаться от конкурентов — **функциональные** и **эмоциональные** преимущества отдельно
9. Смысловая территория бренда (забота / технологичность / простота / статус / скорость / экспертность / близость и т.д.)
10. RTB — конкретные факты, цифры, регалии, подтверждающие преимущества
11. Positioning statement — суть бренда / дескриптор одним предложением `[вывод]`
12. Консистентность коммуникации по каналам — таблица + расхождения

## Правила синтеза

- Каждое утверждение — с цитатой или ссылкой на источник
- Выводы помечать `[вывод]`, факты — без пометки
- Если данных мало — прямо писать «данных недостаточно», не выдумывать
- Финальный файл самодостаточен: читается без открытия трёх исходников
