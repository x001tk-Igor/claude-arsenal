# Сегментация аудитории: выбор метода и гипотезы

Скилл ведёт пользователя через 4 шага: сбор контекста → внутренний скоринг методов → предложение методов с гипотезами сегментов → рекомендации.

База методов (28 шт.) с параметрами и оценками Ч/Р/Д: [METHODS.md](METHODS.md). Загрузить перед шагом 2.

---

## Шаг 1 — Сбор контекста

Задать пользователю вопросы (все обязательные):

1. **Продукт/сервис** — что это?
2. **B2B или B2C?**
3. **Цель сегментации** — предложить список, пусть выберет:
   - Запуск нового продукта
   - Маркетинговая стратегия текущего продукта
   - Бренд-стратегия
   - Повышение выручки
   - Повышение LTV текущих клиентов
   - Запуск новой фичи
   - Улучшение UX / пользовательского пути
   - Продуктовая стратегия
4. **Кто будет использовать сегментацию** — какая команда и роль? (маркетолог, продакт, фаундер, sales и т.д.)
5. **Каналы привлечения** — какие планируются?

Все вопросы задать одним сообщением. Дождаться ответов перед переходом к шагу 2.

## Шаг 2 — Внутренний скоринг и предложение методов

### 2а. Внутренний скоринг (НЕ показывать пользователю)

Загрузить [METHODS.md](METHODS.md). Для каждого из 28 методов оценить два критерия:

**Критерий А — Релевантность:**
> «Сегменты, выделенные этим методом, по-разному покупают продукт и по-разному его используют?»

Входные факторы:
- Тип продукта (B2B/B2C, SaaS, e-commerce, контент, услуги)
- Категория (утилитарная vs эмоциональная)
- Цель сегментации
- Каналы привлечения

Примеры:
- Ценности людей почти бесполезны в B2B
- JTBD плохо подходит для эмоциональных покупок (fashion)
- Индустрия — базовый метод для B2B

**Критерий Б — Однородность:**
> «Будут ли люди внутри сегмента вести себя примерно одинаково при покупке и использовании продукта?»

Входные факторы:
- Специфика продукта и рынка
- Насколько параметр реально влияет на поведение

Пример: поколенческая сегментация полезна для дейтинга (одинаковое поведение внутри поколения), но по интересу «хоккей» особое поведение не выделить.

**Фильтр:** методы с высокой релевантностью + высокой/средней однородностью → в основную рекомендацию. Методы со средней релевантностью или средней однородностью → в дополнительные. Отобрать **топ-4 основных** и **4 дополнительных**.

### 2б. Вывод пользователю

Показать все 8 методов в **одной таблице** с двумя блоками и **сквозной нумерацией от 1 до 8**. Блоки разделить строкой-заголовком:

| 3 | ...   | 🟢/🟡/🔴  | 🟢/🟡/🔴    | 🟢/🟡/🔴    |

| | **Дополнительные — тоже можно рассмотреть** | | | |
| 5 | ...   | 🟢/🟡/🔴  | 🟢/🟡/🔴    | 🟢/🟡/🔴    |
| 6 | ...   | 🟢/🟡/🔴  | 🟢/🟡/🔴    | 🟢/🟡/🔴    |

Расшифровка критериев:

**Четкость (Ч)** — насколько сегменты не пересекаются (MECE):
- 🟢 Не пересекаются, полная картина
- 🟡 Частично пересекаются
- 🔴 Сильно пересекаются

**Размерность (Р)** — детальность и гибкость:
- 🟢 Удобна для персонализации маркетинга и продукта
- 🟡 Ограниченно подходит для персонализации
- 🔴 Слишком верхнеуровневая

**Доступность (Д)** — можно ли реально таргетировать:
- 🟢 Легко найти и достать нужных людей
- 🟡 Есть идеи каналов, нет гарантии
- 🔴 Нет каналов, где точно достать ЦА

Для каждого метода кратко объяснить, почему он подходит для этого кейса (1–2 предложения).

**Комментарии по ограничениям:** если у метода есть 🟡 или 🔴 по какому-то критерию, добавить предупреждение с полным названием критерия:
- **Доступность 🟡/🔴** → «Сложно будет выделить этот сегмент для точного таргетинга — нет каналов, где можно гарантированно достать именно этих людей»
- **Размерность 🔴** → «Может не хватить детализации для персонализации маркетинга и продукта — сегментация слишком верхнеуровневая»
- **Размерность 🟡** → «Ограниченная детализация — для глубокой персонализации может потребоваться комбинация с другим методом»
- **Четкость 🔴** → «Сегменты могут сильно пересекаться — один и тот же человек попадёт в несколько сегментов сразу»
- **Четкость 🟡** → «Частичное пересечение сегментов — нужно чётко определить границы при работе с гипотезами»

Комментарии писать конкретно под кейс, не шаблонно. Объяснять последствия ограничения для конкретного продукта/цели.

Каждый метод в таблице нумеровать (1, 2, 3...), чтобы пользователь мог выбрать просто написав номер. Спросить пользователя, какой метод (или комбинацию) выбирает.

## Шаг 3 — Генерация гипотез сегментов

Для каждого выбранного метода сегментации сформулировать **5 гипотез сегментов**.

Формат каждой гипотезы — заголовок с методом, затем сегменты:

```
### Метод: [Название метода] ([уточнение, что именно сегментируем])

**Сегмент 1: «[Короткое название]»**
- **Кто:** [Конкретный профиль — роль, доход, контекст. Не абстрактно, а с деталями]
- **Поведение:** [Как покупают/используют продукт. Конкретные действия, каналы, триггеры покупки, чувствительность к цене]
- **Почему отдельный сегмент:** [1–2 предложения. Чем принципиально отличается поведение от других сегментов. Какая ключевая мотивация/барьер]
```

Требования к качеству гипотез:
- **Конкретность:** указывать цифры (доход, чек, возраст) где возможно
- **Поведенческая разница:** каждый сегмент должен покупать/использовать продукт по-разному
- **Мотивация:** объяснять не только «кто», но и «почему именно так себя ведут»
- **Инсайт:** в «почему отдельный сегмент» давать неочевидный вывод, а не пересказ профиля

Использовать параметры из METHODS.md для конкретизации.

## Шаг 4 — Рекомендации

Завершить анализ блоком рекомендаций:

1. **Приоритет** — какие сегменты проверить в первую очередь и почему
2. **Что валидировать через интервью** — сформулировать 5–7 конкретных вопросов для исследования, привязанных к гипотезам. Типичные направления:
   - Сколько целевая аудитория реально тратит на решение проблемы сейчас и что покупали раньше
   - Какой формат/способ потребления предпочитают
   - Что стало триггером — когда именно начали искать решение
   - Какие конкретно задачи хотят закрыть (для наполнения продукта)
   - Готовность платить: ценовой диапазон и что за эти деньги ожидают
   Вопросы должны быть конкретными под продукт и сегменты, а не абстрактными.
3. **Предупреждения** — подсветить типичные ошибки, если релевантны:
   - JTBD плохо подходит для эмоциональных категорий
   - Персоны часто недоступны для таргетинга
   - Архетипы сложно использовать в performance-маркетинге
   - Теория поколений не доказана научно
