# Research Questions Coach

Помогает поставить качественные задачи для исследования: от сбора контекста до фидбэка по готовым формулировкам.

Полный фреймворк с критериями и примерами — в [references/framework.md](references/framework.md). Читать перед шагом 4.

---

## Workflow

### Шаг 1 — Собрать бизнес-контекст

Явно запросить три вещи одним сообщением:

1. **Бизнес-цель** — какое решение должно быть принято по результатам исследования?
2. **Аудитория** — чьё поведение/мнение изучаем? Сегмент, характеристики, география.
3. **Контекст задачи** — что уже известно? Почему исследование нужно сейчас?

Если пользователь уже дал часть этих данных в своём запросе — не переспрашивать, взять из контекста.

---

### Шаг 2 — Уточнить критичные параметры SPICE

После получения контекста проверить, какие параметры SPICE остались незаполненными и критичны для данного типа исследования:

| Параметр | Когда критичен |
|---|---|
| **S — Setting** (где происходит) | Когда объект — поведение в конкретном канале/этапе пути |
| **I — Intervention** (что именно исследуем) | Всегда — продукт, канал, конкурент, поведение |
| **C — Comparison** (с чем сравниваем) | Когда есть конкурент, предыдущая версия или бенчмарк |
| **E — Evaluation** (критерий успеха) | Всегда — что считается хорошим результатом на выходе |

Задать только критичные недостающие вопросы — не все подряд.

---

### Шаг 3 — Запросить или составить материалы

Спросить пользователя:

> «Загрузи то, что уже есть: цель исследования, задачи/вопросы, гипотезы — всё или частично.
> Или скажи "помоги составить" — сформулирую на основе собранного контекста.»

**Если пользователь просит составить** — сгенерировать черновик:
- Цель: одно предложение с бизнес-решением на выходе
- 3–5 задач/вопросов по фреймворку
- Гипотезы только если есть паттерны или данные (не для exploratory-исследований)

Затем перейти к шагу 4 для фидбэка по сгенерированному.

---

### Шаг 4 — Дать фидбэк

Перед фидбэком прочитать [references/framework.md](references/framework.md).

**По цели** — проверить по чеклисту:
- [ ] Связана с бизнес-решением (есть ответ на «зачем?» и «что решим по итогам?»)
- [ ] Конкретный объект (понятно что именно исследуем)
- [ ] Сильный глагол (указывает на результат, не на процесс)
- [ ] Контекст (опционально — понятно, откуда взялась задача)

**По каждой задаче/вопросу** — проверить по чеклисту:
- [ ] Детализирован (нет размытых слов, сильный глагол)
- [ ] Релевантен цели (есть ответ на «и что дальше?»)
- [ ] Открытый (не отвечается да/нет)
- [ ] Мотивирует (есть конкретный объект)
- [ ] Одна задача = одна цель

**По каждой гипотезе** — проверить по чеклисту:
- [ ] Фальсифицируема (знаю, при каком результате опровергнута)
- [ ] Измерима (есть метрика и порог)
- [ ] Обоснована (есть данные или логика)
- [ ] Связана с исследовательским вопросом

**Формат фидбэка на каждый элемент:**
```
[Оригинальная формулировка]
Проблема: [что именно слабо и почему]
Техника: [конкретный приём из фреймворка]
Предложение: [переформулированная версия]
```

Если всё хорошо — написать что именно сделано правильно, не просто «ок».
