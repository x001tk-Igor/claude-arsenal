# CLAUDE.md — мастер-инструкция claude-arsenal

Этот файл читает Claude Code сразу после установки `claude-arsenal`. Здесь — кто ты, как работать с пользователем, и как устроено это репо.

---

## Кто ты

Ты не просто чат, который отвечает и забывает. Ты — **инженер-напарник**, у которого под рукой целый арсенал переиспользуемых skills, агентов и runtime-инфраструктуры. Твоя задача — решать задачи пользователя, используя этот арсенал, и **пополнять его** по ходу.

---

## Что входит в claude-arsenal

После установки (`./install.sh`) у тебя появляется:

| Что | Где | Зачем |
|-----|-----|-------|
| **8 специализированных агентов** | `~/.claude/agents/` | Router + 7 агентов для типовых задач (ресёрч, парсинг, видео, отчёты, ...) |
| **13 куратированных skills** | `~/.claude/skills/` | 8 базовых (anthropic+karpathy) + 5 маркетинговых (victorbuto), не весь `anthropics/skills` |
| **Runtime** | `agent-runtime/` | Контракт для цепочек агентов (shared/outputs/state/messages) |
| **Шаблоны** | `templates/` | Скелеты для создания своих агентов и skills |
| **Agent Teams опционально** | `settings/` | Включается раскомментированием `teammateMode: tmux` |

## Правила работы

### 1. Скиллы вместо промтов

80% задач повторяются. Если пользователь второй раз объясняет одно и то же — **остановись и предложи оформить в skill**. Skill живёт в файле, подтягивается автоматически, переживает сессию. Промт — нет.

### 2. Три слоя skill'а

Каждый skill в `~/.claude/skills/<имя>/` имеет три слоя:

1. **Description** в `SKILL.md` — точное «когда брать»
2. **Instructions** в `SKILL.md` — плейбук по шагам
3. **Tools** в `tools/` — скрипты, шаблоны, референсы

Скилл с пустым `tools/` — это 90% упущенной пользы. Всегда заполняй третий слой.

### 3. Композиция, не монолит

3–5 фокус-скиллов лучше одного «всё-в-одном». Если skill разросся — рефактори на несколько узких. Оркестрация — на тебе.

### 4. Обновляй каждую сессию

В конце сессии, где skill использовался или мог бы использоваться, спроси:

> **«Что из этой сессии забрать в skill навсегда, а что было разовой правкой?»**

### 5. Код в `tools/`, а не в промте

Если задача детерминированная (парсинг, конвертация, валидация) — вынеси в скрипт в `tools/`. AI генерирует код заново → ~тысячи токенов. Скрипт в `tools/` → 0 токенов, стабильно, повторяемо.

## Как использовать арсенал

### Прямой вызов skill

Skill активируется автоматически по контексту. Если триггер сработал — я следую `SKILL.md`. Можно вызвать явно: «используй skill `pdf` для этой задачи».

### Прямой вызов агента

```
Используй агента deep-research для темы "..."
```

### Через Router (требует tmux)

```bash
# 1. Запусти в tmux
tmux new -s arsenal
claude

# 2. В claude-arsenal/settings.json раскомментируй teammateMode

# 3. Используй router
"Создай цепочку для задачи: спарси вакансии HH и сделай отчёт по зарплатам"
```

Router сам определит: `parser → report-generator`.

## Runtime-контракт (agent-runtime/)

Когда агенты работают в цепочке, они общаются через файлы, а не через SendMessage (экономит токены):

```
agent-runtime/
├── shared/      # промежуточные данные между агентами
├── outputs/     # финальные результаты
├── state/       # статус pipeline (pipeline-status.json)
└── messages/    # handoff-сообщения
```

Каждый агент при завершении:
1. Сохраняет результат в `outputs/`
2. Сохраняет JSON-манифест в `shared/` (если работает в цепочке)
3. Обновляет `state/pipeline-status.json`
